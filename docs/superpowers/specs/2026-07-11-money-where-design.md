# 「钱都去哪了」专业化重构设计

日期：2026-07-11  
产品名：钱都去哪了  
作者：tphu

## 1. 产品定位

「钱都去哪了」是一款面向个人用户的本地账单分析工具。用户导入银行卡、微信支付和支付宝账单后，工具在浏览器中完成解析、跨来源去重、资金流分类和可视化分析，回答三个核心问题：钱从哪里来、钱真正花到哪里、接下来最值得关注什么。

主产品保持零安装和本地优先：打开 `index.html` 即可使用，账单内容不由本工具上传。第三方解析与图表库随项目本地提供，运行时不再加载远程脚本。旧文件 `share_bill_analyzer.html` 作为兼容入口跳转到新首页。

## 2. 设计目标与非目标

### 目标

- 将产品名统一为「钱都去哪了」，作者统一为 `tphu`。
- 用真实、清晰的支付宝与微信支付品牌图标替代 emoji。
- 修正跨来源重复、内部转账、投资申购、信用还款和退款混入口径的问题。
- 将分析从“展示几个总数”升级为可解释的数据质量、消费结构、变化趋势、异常与行动建议。
- 消除硬编码姓名、PDF 密码、真实账单路径及动态 HTML 注入风险。
- 建立可重复运行的测试与明确的隐私使用说明。

### 非目标

- 不建设服务器、账号系统、云端同步或多人协作。
- 不提供证券、基金或信贷产品推荐，不承诺收益，不替代专业财务建议。
- 不尝试一次性支持所有银行版式；银行 PDF/CSV 使用可解释的通用解析，并明确显示解析质量。
- 不删除或移动用户现有的真实账单文件，只通过忽略规则和文档阻止误分享。

## 3. 技术架构

产品采用无框架的静态 Web 架构，减少安装和运行负担：

```text
index.html
├── styles.css                 视觉系统、响应式与可访问性
├── src/core.js                纯函数：归一化、分类、去重、统计、建议、安全转义
├── src/app.js                 文件选择、平台解析、状态管理、DOM 与图表渲染
├── assets/brands/*.svg        支付宝、微信支付本地品牌图标
└── vendor/*                   固定版本的本地解析/图表库

tests/core.test.js             分析口径与去重行为
tests/structure.test.js        品牌、语义结构和安全约束
```

`src/core.js` 使用 UMD 导出，浏览器通过 `window.BillAnalyzerCore` 使用，Node 测试通过 `require()` 使用。业务规则只在这一处实现。`src/app.js` 只负责平台文件解析和界面协调，不复制分类或统计规则。

Python 文件保留为兼容工具，但移除真实姓名、密码和固定文件路径，改为命令行参数；文档明确新 Web 入口是推荐使用方式。

## 4. 统一交易模型

每笔标准交易至少包含：

- `id`：稳定的本地记录标识；
- `source`：`bank`、`wechat`、`alipay`；
- `date`、`time`、`datetime`；
- `amount`：正数金额；
- `direction`：原始方向 `income`、`expense`、`neutral`；
- `flowType`：专业资金流口径；
- `categoryId`、`categoryName`、`needType`；
- `merchant`、`description`、`paymentMethod`、`status`；
- `searchText`；
- `confidence`、`ruleId`：分类置信度和命中依据；
- `matchedSource`：跨来源去重关联；
- `warnings`：解析或分类疑点。

`flowType` 分为六类：

1. `income`：工资、奖金等真实可支配收入；
2. `expense`：真实消费；
3. `refund`：消费退款，用于冲减消费而不是新增收入；
4. `transfer`：本人账户、余额工具、小荷包之间的内部流转；
5. `asset`：基金、理财申购赎回等资产形态变化；
6. `debt`：信用卡、花呗、白条等还款，不与已记录消费重复计算。

消费再按 `needType` 标记为 `essential`（必要）、`discretionary`（可选）或 `unclassified`（待确认）。渠道是支付方式，不再作为消费分类。

## 5. 解析、状态过滤与去重

- 支付宝 CSV 必须使用标准 CSV 解析器，正确处理 GBK/UTF-8、引号和字段内逗号。
- 微信 XLSX 和支付宝 CSV 读取交易状态；明确排除失败、关闭或无效记录，退款保留为 `refund`。
- 相同来源的“同日、同商户、同金额”记录不能直接删除，因为它们可能是真实重复购买。
- 跨来源只合并“银行卡摘要明确指向微信/财付通/支付宝”且日期、金额兼容的记录。
- 双方都有时间时使用小时间窗匹配；缺失时间时，仅在候选唯一且渠道相符时合并，否则保留并加入疑点。
- 分析结果展示原始条数、有效条数、跨来源匹配数、忽略状态数和待确认条数，允许用户判断可信度。

## 6. 指标与分析维度

### 核心指标

- 真实收入：仅 `income`；
- 净消费：`expense` 合计减去 `refund`；
- 现金结余：真实收入减净消费；
- 结余率：现金结余 / 真实收入；收入为零时显示“暂无口径”，而不是 0%；
- 日均消费：按覆盖日期的包含首尾天数计算；
- 内部流转、资产变动、债务偿还单独展示，不计入消费。

### 专业分析

- 消费分类排名：金额、占比、笔数、客单价；
- 必要与可选消费结构；
- 月度收入、净消费、结余趋势，并标记不足整月的数据；
- 最近完整月与此前完整月的变化及前三个驱动分类；
- 商户集中度和高频小额消费；
- 周期性消费候选：同时考虑出现月份、金额稳定性和时间间隔，不直接称为“订阅”；
- 大额与异常消费：依据个人历史分布生成阈值，不使用固定 1000 元作为唯一标准；
- 未分类率、解析警告和数据来源覆盖。

## 7. 建议引擎

建议采用“发现 → 证据 → 影响 → 可选行动”结构，并按数据质量、现金流风险、异常变化、可优化支出、表现稳定排序。

建议必须满足：

- 每条建议引用实际金额、占比、变化或笔数；
- 数据不足时明确说明，避免强结论；
- 不推荐具体基金、平台、借贷产品或固定资产配置比例；
- 不把结余率简单贴为“优秀/糟糕”，而是结合覆盖月份和收入稳定性；
- 行动以预算提醒、核对分类、检查周期支出、建立应急缓冲等低风险选择为主；
- 页面注明“结果仅用于个人记账参考，不构成投资、信贷或税务建议”。

## 8. 界面与品牌设计

视觉方向为“现代账本 / 编辑部数据报告”：暖白纸张底色、墨色正文、财务绿主色、琥珀色提醒。标题使用中文宋体风格形成可信、有人味的编辑感；正文使用清晰黑体；金额采用等宽数字。避免蓝紫渐变、重阴影、满屏水印和 emoji 图标。

首页结构：

1. 顶部品牌与隐私说明；
2. 一句核心价值“把分散账单，整理成一张清楚的钱流地图”；
3. 银行卡、微信支付、支付宝三张语义化上传卡；
4. 文件状态、PDF 密码和开始分析按钮；
5. 支持格式、数据不上传和本地依赖说明。

报告结构：

1. 报告头与数据覆盖/质量条；
2. 净消费、真实收入、现金结余、结余率四项核心指标；
3. “钱去了哪里”横向分类条形图与必要/可选结构；
4. 月度收支趋势；
5. 关键发现与可选行动；
6. 支付渠道、周期性候选和大额/异常交易；
7. 可筛选的完整明细与返回重新分析。

微信支付和支付宝使用本地 SVG 品牌图标，保留品牌原色、比例和留白；图标旁始终显示平台名称。其余功能图标使用统一线性 SVG，不使用 emoji。

## 9. 可访问性与响应式

- 上传使用原生 `label`、`input`、`button`，全部功能可键盘操作；
- 所有输入均有可见标签，状态区域使用 `aria-live`；
- 交互元素有 44×44 像素触控目标和明显的 `:focus-visible`；
- 图表提供文字摘要或表格替代，不只依赖颜色；
- 正文和背景对比度达到 WCAG AA；
- 遵守 `prefers-reduced-motion`；
- 在 375、768、1024、1440 像素宽度验证；手机端明细采用紧凑卡片/横向可控布局，不破坏视口。

## 10. 隐私与安全

- 运行时不加载远程脚本或字体；
- 动态账单文本使用 DOM `textContent` 或统一转义，不直接拼接未经处理的 `innerHTML`；
- 不把交易数据写入 localStorage、日志或自动生成文件；刷新页面即清空；
- PDF 密码只存在当前输入框和解析调用中，不保存；
- `.gitignore` 屏蔽 PDF、XLS/XLSX、真实 CSV、生成 JSON/HTML 和临时解密文件；
- README 明确现有目录含真实账单，分享前必须只分享产品源码或脱敏样例；
- 不删除用户现有数据，由用户自行归档。

## 11. 验收标准

- 页面所有旧作者文案均替换为 `tphu`，产品名为「钱都去哪了」；
- 微信支付和支付宝在上传、来源概览和交易明细中使用对应品牌图标；
- 失败/关闭交易不计入，退款冲减消费，内部转账、投资、还款不计入真实消费；
- 同日同商户同金额的两笔真实消费不会被同源误删；
- 跨来源唯一匹配可去重，存在歧义时保留并显示警告；
- 建议不包含具体金融产品或固定投资比例；
- 核心单元测试全部通过，结构测试覆盖品牌、远程依赖与可访问性关键项；
- 页面在桌面和手机尺寸无阻断性布局问题，浏览器控制台无运行错误；
- `index.html` 可作为最终入口，`share_bill_analyzer.html` 可兼容跳转。
