# 「钱都去哪了」单 HTML 完整离线版设计

## 目标

生成一个可直接分享的 `钱都去哪了-离线版.html`。好友把文件保存到手机或电脑后，使用 Safari、Chrome、Edge 等浏览器打开，即可导入并分析账单；不需要安装程序、运行脚本、申请域名、连接服务器或使用 VPN。

## 功能范围

- 保留现有三类导入：银行卡 PDF、微信支付 XLSX、支付宝 CSV。
- 保留现有完整报告：收支概览、退款和内部流转区分、分类与趋势图、消费明细、跨来源对账以及专业建议。
- 保留支付宝与微信支付品牌图标、响应式页面和打印功能。
- 在线版继续保留；离线文件作为最简单的好友分享方式。

## 文件结构与构建方式

- 源代码继续按当前 HTML、CSS、JavaScript 和本地第三方依赖分文件维护。
- 新增确定性的离线构建脚本，把页面样式、业务代码、图表库、CSV/XLSX/PDF 解析库和图标内联到一个 HTML 文件。
- PDF.js Worker 代码内嵌在文件中，运行时只转换为浏览器内存中的 Blob URL，避免 `file://` 环境下因找不到外部 Worker 而导致银行卡 PDF 解析失败。
- 第三方许可证与商标说明作为不可执行的内嵌文本随文件分发，保证单文件分享时许可信息不丢失。
- 构建产物只包含公开运行代码和静态资源，不包含任何账单、分析缓存、密码或本机路径。

## 隐私与安全

- 文件不包含统计、广告、远程字体、CDN 或第三方接口。
- 内容安全策略禁止网络连接，只允许当前文件内的脚本、样式、图片和内存 Worker 运行。
- 用户选择的账单仅在当前浏览器内存中解析；刷新或关闭页面后不保留。
- 页面文案准确表述为“账单内容不上传”；浏览器或操作系统自身的文件访问记录不属于本工具收集的数据。

## 手机打开体验

- 文件名固定为 `钱都去哪了-离线版.html`，页面显示“完整离线版”标识。
- 页面提供简短提示：若微信、QQ 或系统文件预览器不执行 HTML，请先保存文件，再选择 Safari、Chrome、Edge 或其他浏览器打开。
- 在 375px 宽度下保持上传区、图表、筛选器和交易列表无横向溢出、遮挡或重叠。
- 不承诺所有聊天软件的内置预览器都能执行 JavaScript；外部浏览器打开是受支持路径。

## 错误处理

- 浏览器禁用 JavaScript 时，显示可读提示，说明需要改用支持 JavaScript 的浏览器。
- 单个账单格式不受支持、文件损坏或 PDF 密码错误时，继续使用现有的明确错误提示，不影响其他已选择账单。
- 若浏览器不支持 Blob Worker，银行卡 PDF 显示兼容性提示；微信和支付宝导入仍可使用。

## 验收标准

1. 最终交付只有一个 HTML 文件，双击或从浏览器打开即可运行。
2. HTML 不引用任何相对或远程脚本、样式、图标、字体和 Worker 文件。
3. 浏览器运行期间除打开该 HTML 本身外不产生网络请求。
4. 示例数据可完整生成报告，桌面端和 375px 手机视口无重叠或横向滚动。
5. 支付宝真实账单回归测试仍提取 675 条有效记录；测试过程不把该账单复制进发布文件。
6. 微信 XLSX、支付宝 CSV 和一份无隐私的测试 PDF 均可在单文件环境完成解析。
7. 自动测试验证单文件不包含 CSV、XLSX、PDF 账单内容、个人身份信息或本机绝对路径。
8. 原有自动测试全部通过，并新增离线构建、网络隔离和 PDF Worker 回归测试。

## 非目标

- 不把账单保存到云端或浏览器持久存储。
- 不提供账号、同步、多人共享或服务器端解析。
- 不依赖域名解决本次分享需求；域名和国内托管可作为后续在线版增强单独处理。
