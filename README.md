# 钱都去哪了

作者：`tphu`

「钱都去哪了」是一个本地优先的个人账单分析工具。它在当前浏览器中解析银行卡、微信支付和支付宝账单，区分真实消费、退款、内部流转、资产变动和债务偿还；不会要求账号，也不会上传账单。

## 最适合分享朋友的方式

在项目目录运行：

```bash
npm run build:offline
```

会生成 [钱都去哪了-离线版.html](output/钱都去哪了-离线版.html)。把这一个文件发给朋友即可。

- 电脑端请先保存文件，再用 Safari、Chrome 或 Edge 打开；这是推荐且更可靠的方式。
- 移动端（Android 等）能否运行本地 HTML，取决于系统和接收应用；可尝试支持本地 HTML 的浏览器或 HTML 查看器。
- iPhone/iPad 的“文件”或聊天预览不一定会执行本地 JavaScript；若只能预览或控件无法使用，请改用兼容浏览器/查看器；如果没有兼容应用，请在电脑打开。

无论文件如何分享，账单始终只在打开文件的那台设备上处理。

## 使用方法

1. 选择银行卡 PDF/CSV、微信 XLS/XLSX、支付宝 CSV；每类可一次选多个文件。
2. 点击“开始分析”。文件会按顺序在浏览器中解析并合并去重。
3. 在“总览”查看消费、收入、结余、分类、月度变化和预算；在“消费画像”查看时段、商家与餐饮场景。
4. 如需留存，点击“导出 Excel”，确认后会下载本地工作簿。

预算只保存到当前浏览器会话；关闭会话后会自动清除，账单内容不会被保存。

## Excel 导出内容

导出的 `.xlsx` 包含六张表：

- `总览`：范围、收入、净消费、结余、结余率与预算摘要；
- `月度趋势`：月度收入、消费、退款、结余与环比；
- `分类分析`：金额、占比、笔数、客单价和消费属性；
- `商家分析`：次数、总额、客单价、最近日期和主要分类；
- `交易明细`：统一后的交易记录；
- `数据质量`：读取、忽略、关联和警告统计。

交易明细属于敏感数据，导出前请确认接收人和保存位置可信。

## 本地开发

macOS 可双击 [启动工具.command](tools/启动工具.command)，或运行：

```bash
python3 tools/start.py
```

随后打开终端显示的 `127.0.0.1` 地址。CSV/XLSX 和示例通常可直接打开 `index.html`；PDF.js 的 Worker 在部分浏览器的 `file://` 策略下会受限，因此开发和 PDF 解析建议使用本地启动器。

## 支持的文件与口径

- 银行卡：带文字层和标准表头的 PDF，或 CSV；
- 微信支付：官方导出的 XLS/XLSX；
- 支付宝：官方导出的 CSV，兼容 GB18030/UTF-8、引号和字段内逗号。

净消费 = 真实消费 − 退款。内部流转、资产变动和债务偿还不会计入“钱真正花掉了多少”，但会单独展示。跨平台候选不唯一时不会自动删除，并会在数据质量中说明。

消费画像和建议只在数据量、带时间交易和日期跨度足够时生成；每条建议都包含实际数据依据和一个可选择的行动，不推荐具体理财、借贷或信用产品。

## 隐私与安全

所有账单内容只在浏览器内存中处理。运行时不请求网络、不加载远程脚本或字体；离线版的 CSP 明确禁止网络连接。PDF 密码仅在本次解析内存中使用。Excel 文本会做公式注入防护。

真实账单、报告和解密文件不应上传到公开仓库或公开网盘。`.gitignore` 默认屏蔽常见账单和生成物；发布网页时只发布 `npm run build` 生成的 `dist/`，不要上传整个项目目录。

## 项目结构

```text
index.html                    主入口
styles.css                    视觉系统、主题和响应式
src/core.js                   统一解析、分类、去重与基础财务分析
src/insights.js               消费画像与可解释建议
src/budget.js                 会话级预算
src/exporter.js               Excel 导出与公式注入防护
src/app.js                    文件队列、页面交互和图表
tools/                        本地启动与兼容 Python 工具
scripts/build-static.js       静态发布包构建器
scripts/build-offline.js      单 HTML 离线构建器
```

第三方库版本、许可证和品牌图标来源见 `vendor/README.md`。
