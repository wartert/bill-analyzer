# 钱都去哪了

作者：`tphu`

一款本地优先的个人账单分析工具。导入银行卡、微信支付和支付宝账单后，它会在当前浏览器中完成解析、跨来源去重、资金流分类与可视化，帮助你区分真实消费、退款、内部流转、投资申购和债务偿还。

## 最简单的分享方式：单 HTML 离线版

维护者在项目目录运行：

```bash
npm run build:offline
```

生成的 `钱都去哪了-离线版.html` 位于项目根目录。接收者保存文件后按设备选择打开方式：

- 电脑：使用 Safari、Chrome 或 Edge 打开，这是推荐且更可靠的方式；
- Android 和其他移动端：能否运行本地 HTML 取决于操作系统和接收应用。可以尝试支持本地 HTML 的浏览器或 HTML 查看器，但聊天应用和文件预览器可能不会执行 JavaScript；
- iPhone/iPad：“文件”和聊天应用的预览不一定会可靠执行任意本地 HTML；如果没有兼容应用，请在电脑上打开。

成功打开后，选择银行卡 PDF、微信 XLSX 或支付宝 CSV。若页面只能预览或控件无法使用，请改用支持本地 HTML 的浏览器或 HTML 查看器；没有兼容应用时，请在电脑上打开。账单在当前浏览器中本地处理，不会上传。

## 快速开始

正式分享版本是纯静态网页。朋友通过 HTTPS 链接打开后即可在手机或电脑上使用，不需要安装 Python，也不需要运行本机脚本。页面文件虽然托管在网站上，但账单解析仍发生在访问者自己的浏览器内，账单不会上传到托管平台。

本地开发时，macOS 可以双击 `启动工具.command`，也可以在此目录运行：

```bash
python3 start.py
```

随后打开终端显示的本机地址。CSV/XLSX 和脱敏示例通常也能直接打开 `index.html` 使用；PDF.js 的 Worker 受部分浏览器的 `file://` 安全策略限制，因此完整功能推荐使用上述本机启动方式。

## 支持的文件

- 银行卡：带文字层和标准表头的 PDF，或 CSV；
- 微信支付：官方导出的 XLSX；
- 支付宝：官方导出的 CSV，支持 GB18030/UTF-8、引号和字段内逗号。

三个来源可以单独导入，也可以一起导入。一起导入时，工具会按渠道、日期、金额和时间关联银行卡与微信/支付宝的同一笔记录；候选不唯一时不会自动删除，并会在数据质量中显示疑点。

## 数据口径

- 真实收入：工资、奖金、劳务等可支配收入；
- 净消费：真实消费减退款；
- 内部流转：本人账户、微信零钱、余额工具等之间的资金移动；
- 资产变动：基金、理财等申购与赎回；
- 债务偿还：信用卡、花呗、白条等还款。

内部流转、资产变动和债务偿还不会计入“钱真正花掉了多少”，但会单独展示。扫码付款等缺少商户语义的记录会保留为不确定分类，避免制造虚假的精确结论。

## 隐私与安全

所有账单内容只在浏览器内存中处理。本工具不会把交易写入 `localStorage`，不会主动上传账单，也不会在运行时加载远程脚本或字体；刷新页面后内存数据即清空。

项目目录中现有的 PDF、XLSX、CSV、JSON 和旧报告可能包含真实姓名、账号、商户及交易明细：

- 不要把整个目录上传到公开仓库、网盘公开链接或发送给他人；
- `.gitignore` 已默认屏蔽账单、解密文件和带明细的生成报告；
- 不要直接上传整个项目目录；运行 `npm run build` 后，只发布自动生成的 `dist/`；
- 本次重构不会自动删除或移动你的原始账单，请自行决定安全归档位置。

分类和建议基于账单文本推断，仅用于个人记账参考，不构成投资、信贷、税务或其他专业建议。

## 验证

```bash
npm run build
npm test
python3 -m py_compile start.py analyze_all.py parse_zhongyuan.py analyze.py
```

构建脚本采用文件白名单，只会把网页运行所需文件和第三方许可证复制到 `dist/`，不会包含 PDF、CSV、XLS/XLSX、Python 脚本或分析结果。测试覆盖资金流分类、退款、去重歧义、同源真实重复、平台状态、混合换行的支付宝 CSV、旋转 PDF 表格、HTML 转义、图表布局时序和发布包隐私边界。
